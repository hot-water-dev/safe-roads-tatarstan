# Анализ аварийности в Республике Татарстан
> Сделано за ночь до сдачи

## Описание проекта
Интерактивная система анализа и визуализации аварийности на дорогах Республики Татарстан. 
Цель — выявить очаги концентрации ДТП со смертельным исходом и сформировать рекомендации по снижению аварийности.

## Данные
**Источник:** Официальный портал stat.gibdd.ru  
**Тип данных:** Выгрузки в формате **XML** за 2025–2026 годы по Республике Татарстан

## Используемые технологии
- **Backend:** Python – FastAPI, Pandas, Scikit-learn (DBSCAN), OSMnx
- **Database:** SQLite
- **Frontend:** JavaScript – React, TypeScript, Leaflet
- **AI:** YandexGPT API / llm7.io

## Планируемые фичи
1. **Интерактивная карта** ДТП по Республике Татарстан
    - ☆ Граф дорог с возможностью панорамирования и масштабирования (*OpenStreetMap*)
    - ☆ Маркеры кластеров ДТП — каждый маркер представляет группу аварий в одном месте
    - Цветовое кодирование по тяжести последствий
    - Таймлайн-ползунок — интерактивный слайдер для анализа динамики по месяцам
    - ☆ Попапы — при клике на маркер открывается карточка со статистикой кластера

2. **Система оценки опасности:**
        - **critical** — есть погибшие
        - **high** — больше 5-ти раненых
        - **medium** — больше 2-ух раненых
        - **low** — менее 2-ух раненых
   
4. **Автоматический поиск очагов** — выявление мест с аномальной плотностью ДТП
    - Кластеризация DBSCAN (*Scikit-learn*) по категориям:
        - ☆ География: широта, долгота (увеличенный радиус для трасс)
        - Тип ДТП: столкновение, наезд на пешехода и т.д.
        - Тяжесть последствий: количество погибших и раненых
        - Временной фактор: время суток (светлое/тёмное)
        - Дорожные условия: категория дороги, состояние проезжей части, освещение

5. **Отчётность** — аналитика для принятия решений
    - ☆ Антирейтинг самых опасных участков трасс
    - ☆ Общая статистика: количество ДТП, погибших, раненых, типы ДТП
    - Факторный анализ: поиск корреляций
    - Ссылки на панорамы Яндекс/2ГИС в карточках очагов

6. ☆ **ИИ-анализ** — LLM генерирует текстовые рекомендации для каждого очага на основе статистики
