# Анализ аварийности в Республике Татарстан
> Сделано за ночь до сдачи

## Описание проекта
Интерактивная система анализа и визуализации аварийности на дорогах Республики Татарстан. 
Цель — выявить очаги концентрации ДТП со смертельным исходом и сформировать рекомендации по снижению аварийности.

## Данные
**Источник:** Официальный портал stat.gibdd.ru.
**Тип данных:** Выгрузки в формате **XML** за 2025-2026 годы по Республике Татарстан.

## Используемые технологии
- **Backend:** Python – FastAPI, Pandas, Scikit-learn (DBSCAN), OSMnx.
- **Database:** SQLite.
- **Frontend:** JavaScript – React, TypeScript, Leaflet.
- **AI:** YandexGPT API / llm7.io.

## Планируемые фичи
1. **Интерактивная карта** ДТП по Республике Татарстан.
    - ☆ Граф дорог с возможностью панорамирования и масштабирования (*OpenStreetMap*).
    - ☆ Маркеры кластеров ДТП — каждый маркер представляет группу аварий в одном месте.
    - Цветовое кодирование по тяжести последствий:
        - **Зелёный** — нет погибших
        - **Жёлтый** — есть погибшие
        - **Красный** — погибло более 50% участников ДТП
    - Таймлайн-ползунок — интерактивный слайдер для анализа динамики по месяцам.
    - ☆ Попапы — при клике на маркер открывается карточка со статистикой кластера.

2. **Автоматический поиск очагов** — выявление мест с аномальной плотностью ДТП.
    - Кластеризация DBSCAN (*Scikit-learn*) по категориям:
        - ☆ География: широта, долгота (увеличенный радиус для трасс)
        - Тип ДТП: столкновение, наезд на пешехода и т.д.
        - ☆ Тяжесть последствий: количество погибших и раненых
        - Временной фактор: время суток (светлое/тёмное)
        - Дорожные условия: категория дороги, состояние проезжей части, освещение

3. **Отчётность** — аналитика для принятия решений.
    - ☆ Антирейтинг самых опасных участков трасс.
    - ☆ Общая статистика: количество ДТП, погибших, раненых, типы ДТП.
    - Факторный анализ: поиск корреляций.
    - Ссылки на панорамы Яндекс/2ГИС в карточках очагов.

4. ☆ **ИИ-анализ** — LLM генерирует текстовые рекомендации для каждого очага на основе статистики.

## План реализации
1. **Этап 1 (Подготовка данных):** Получение XML с stat.gibdd.ru, загрузка в SQLite.
2. **Этап 2 (Backend):** Реализация API на FastAPI, кластеризация DBSCAN, расчёт статистики.
3. **Этап 3 (Frontend):** Разработка на React — интерактивная карта, фильтры.
4. **Этап 4: (Интеграция ИИ)** Подключение LLM для анализа кластеров и генерации рекомендаций.
5. **Этап 5 (Тестирование и подготовка к презентации проекта)** Тестирование, запись видеопрезентации, подготовка отчёта.

## Детализация плана

### Этап 1: Подготовка данных (3 июля)
- Получить XML-выгрузки с stat.gibdd.ru за 2025-2026 гг.
- Извлечь данные из вложенной структуры документа.
- Привести данные к единому формату.
- Сохранить данные в локальную БД SQLite.
**Технологии:** Python - Lxml, Pandas, Sqlite3.
**Результат:** SQLite-база.

### Этап 2: Backend (4-5 июля)
- Организовать FastAPI-сервер с подключением к SQLite.
- Реализовать эндпоинты.
- Настроить кластеризацию DBSCAN по координатам и тяжести последствий.
- Сформировать отчетность для принятия решений.
**Технологии:** Python - FastAPI, Scikit-learn, Pandas, Numpy, Sqlalchemy.
**Результат:** REST API.

### Этап 3: Frontend (6-7 июля)
- Развернуть React-приложение с картой OpenStreetMap.

- Реализовать попапы со статистикой.
**Технологии:** JavaScript - React, TypeScript, React-leaflet.
**Результат:** Интерактивная карта с цветными маркерами, попапами и таймлайном.

**Технологии:** Python - FastAPI; API - YandexGPT, llm7.io.
**Результат:** Карта с ИИ-рекомендациями в попапах.
