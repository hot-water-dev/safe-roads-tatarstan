# Анализ аварийности в Республике Татарстан (на примере Казани)

## Описание проекта
Интерактивная система анализа и визуализации аварийности на дорогах Казани. 
Цель — выявить очаги концентрации ДТП и сформировать рекомендации по снижению аварийности.

## Данные
В проекте используются **официальные выгрузки списков карточек ДТП** с портала stat.gibdd.ru (формат XLS/CSV). Для получение конкретных GPS-координат применяется технология **геокодирования адресов** (Yandex Geocoder).

## Планируемые фичи
1. **Интерактивная карта** ДТП по Казани (тепловая карта и маркеры).
2. **Автоматический поиск очагов** (кластеризация DBSCAN) — выявление мест с аномальной плотностью ДТП.
3. **Антирейтинг** самых опасных перекрестков и улиц.
4. **Таймлайн-ползунок** — анализ динамики аварий по месяцам.
5. **ИИ-анализ** — LLM генерирует текстовые рекомендации для каждого очага на основе статистики (тип ДТП, тяжесть).

## Стек технологий
- **Backend:** Python – FastAPI, Pandas, Scikit-learn.
- **Database:** PostgreSQL.
- **Frontend:** JavaScript – React, TypeScript.
- **AI:** YandexGPT API / llm7.io.

## План реализации
1. **Этап 1 (Подготовка данных):** Выгрузка списков ДТП за несколько лет. Очистка данных и загрузка в PostgreSQL.
2. **Этап 2 (Backend):** Реализация API на FastAPI. Настройка алгоритма кластеризации (DBSCAN) для поиска очагов аварийности.
3. **Этап 3 (Frontend):** Разработка React-приложения с интерактивной картой.
4. **Этап 4 (Интеграция ИИ):** Подключение LLM для анализа кластеров и генерации рекомендаций.
5. **Этап 5 (Тестирование и презентация проекта):** Тестирование, запись видеопрезентации, подготовка отчета.

## Детализация плана

### Этап 1: Подготовка данных (1-2 июля)
**Задачи:**
- Скачать официальные выгрузки списков ДТП с stat.gibdd.ru за 2023-2025 годы.
- Очистить привести к единному формату данные.
- Геокодировать адреса через Yandex Geocoder API для получения GPS-координат.
- Загрузить данные в PostgreSQL.
**Технологии:** Python, Yandex Geocoder API, PostgreSQL
**Результат:** Готовая база данных с данными о ДТП, включая координаты, даты, типы аварий и последствия (ранены, погибли, общее количество участников ДТП).

### Этап 2: Backend (3-5 июля)
**Задачи:**
- Реализовать REST API на FastAPI для получения данных.
- Настроить подключение к PostgreSQL.
- Реализовать алгоритм кластеризации DBSCAN для автоматического поиска очагов аварийности.
- Рассчитать статистику для каждого кластера (количество ДТП, последствия, типы).
**Технологии:** FastAPI, Scikit-learn (DBSCAN), Pandas
**Результат:** Работающий API, который возвращает данные о ДТП и кластерах с координатами и статистикой.

### Этап 3: Frontend (6-8 июля)
**Задачи:**
- Разработать React-приложение с интерактивной картой на Leaflet
- Отобразить ДТП в виде маркеров.
- Добавить тепловую карту.
- Реализовать фильтры по типу ДТП, району, дате.
- Добавить таймлайн-ползунок для анализа по месяцам.
- Добавить карточку кластера со статистикой.
**Технологии:** React, TypeScript, Leaflet/React-Leaflet
**Результат:** Интерактивная карта ДТП в городе Казань.

### Этап 4: Интеграция ИИ (9-10 июля)
**Задачи:**
- При клике на кластер фронтенд запрашивает у бэкенда детальную информацию
- Бэкенд формирует промпт для LLM на основе статистики кластера
- LLM генерирует рекомендации по снижению аварийности
- Отобразить рекомендации в карточке кластера
**Технологии:** YandexGPT API / llm7.io, FastAPI
**Результат:** Карточка кластера содержит готовые рекомендации от ИИ.

### Этап 5: Тестирование и презентация проекта (11 - 12 июля)
**Задачи:**
- Протестировать API.
- Проверить работу сайта.
- Записать видеопрезентацию (2-3 минуты) с демонстрацией работы проекта.
- Подготовить отчет по летней практике.
**Технологии:** OBS Studio (запись экрана).
**Результат:** Готовая видеопрезентация, загруженная в репозиторий. Оформленный отчет по летней практике.
